# 06. 比較・まとめ

4 つの移行パターンを、変更量・運用負荷・コスト・モダナイズ度合いで比較します。

## 比較表

| 観点 | Rehost | DB PaaS 化 | コンテナ化 | フル PaaS 化 |
|---|---|---|---|---|
| アプリ変更 | ほぼなし | 小 | 大 | 中〜大 |
| DB 変更 | なし | Azure SQL へ移行 | Azure SQL へ移行 | Azure SQL へ移行 |
| 運用負荷 | 高 | 中 | 低 | 低 |
| スケーラビリティ | 低 | 低〜中 | 高 | 中〜高 |
| モダナイズ度 | 低 | 中 | 高 | 高 |

## 判断の目安

### Rehost が向くケース
- まずは早く Azure に移したい
- 変更リスクを抑えたい

### DB PaaS 化が向くケース
- DB 運用を先に改善したい
- Web アプリは段階的に残したい

### コンテナ化が向くケース
- DevOps / ポータビリティ / 自動スケールを重視する
- アプリ改修に投資できる

### フル PaaS 化が向くケース
- 運用負荷削減を最優先したい
- Azure 標準サービスに寄せたい

## まとめ

この HOL では、**同じ移行元 (`APP01` / `DB01`)** に対して複数のクラウド移行パターンを比較できるようにしています。  
実プロジェクトでは、スケジュール・運用体制・既存コードの状態に応じて最適な方式を選択してください。
