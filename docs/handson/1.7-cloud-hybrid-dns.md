# 1.7. ハイブリッド DNS の構成

> **Note**  
> このページは [`1.1-initial-setup.md`](./1.1-initial-setup.md) の **方法 A「デプロイ後に必要な手動設定」** および **方法 B — Step 5** の詳細手順です。  
> 方法 A で Deploy to Azure を使った場合も、DNS 転送設定は自動化されないため、このページの手順を実行してください。

VPN 接続（[00e](./1.6-cloud-vpn-connect.md)）だけでは、疑似オンプレ側とクラウド側の間で名前解決ができません。  
**Azure DNS Private Resolver** を中心にした双方向の DNS 転送で、VPN 越しの名前解決を実現します。

## 目的

- クラウド側から `lab.local`（Active Directory ドメイン）を解決できるようにする
- 疑似オンプレ側から `privatelink.database.windows.net`（Azure SQL の Private Endpoint）を解決できるようにする

## 前提条件

- [1.6](./1.6-cloud-vpn-connect.md) が完了し、VPN の `connectionStatus` が `Connected` になっている
- Hub 側に DNS Private Resolver（`dnspr-hub`）がデプロイされている
- `az login` 済みであること

### 事前に準備する値

| パラメータ | 既定値 | 説明 |
|-----------|--------|------|
| `OnpremResourceGroup` | `rg-onprem` | 疑似オンプレ リソースグループ名 |
| `HubResourceGroup` | `rg-hub` | Hub リソースグループ名 |
| `OnpremDnsTarget` | `10.0.1.4` | DC01 の IP アドレス（クラウド→オンプレ転送先） |
| `OnpremDomain` | `lab.local` | Active Directory ドメイン名 |

> すべてのパラメータに既定値があるため、既定のまま使う場合は指定不要です。

## 全体像

```
疑似オンプレ (10.0.0.0/16)                    クラウド / Hub (10.10.0.0/16)
┌─────────────────────┐                      ┌──────────────────────────────┐
│  DC01 (10.0.1.4)    │                      │  DNS Private Resolver        │
│  ┌────────────────┐ │   VPN (S2S)          │  ┌────────────┐             │
│  │ DNS Server     │◄├──────────────────────├──┤ Outbound   │             │
│  │                │ │  lab.local の         │  │ Endpoint   │             │
│  │ 条件付き       │ │  クエリを転送         │  └────────────┘             │
│  │ フォワーダー   │─├──────────────────────├─►┌────────────┐             │
│  │                │ │  privatelink.*.net の  │  │ Inbound    │             │
│  └────────────────┘ │  クエリを転送         │  │ Endpoint   │             │
│                     │                      │  │ 10.10.5.4  │             │
└─────────────────────┘                      │  └─────┬──────┘             │
                                             │        │                    │
                                             │  ┌─────▼──────────────────┐ │
                                             │  │ Private DNS Zone       │ │
                                             │  │ privatelink.           │ │
                                             │  │   database.windows.net │ │
                                             │  └────────────────────────┘ │
                                             └──────────────────────────────┘
```

| 方向 | 転送元 | 転送先 | 対象ゾーン | 設定方法 |
|---|---|---|---|---|
| オンプレ → クラウド | DC01（条件付きフォワーダー） | DNS Private Resolver（インバウンド `10.10.5.4`） | `privatelink.database.windows.net` | `az vm run-command`（スクリプト） |
| クラウド → オンプレ | DNS Private Resolver（アウトバウンド） | DC01（`10.0.1.4`） | `lab.local` | DNS Forwarding Ruleset（スクリプト） |

## 手順

### 方法 1: スクリプトで一括実行（推奨）

`Setup-HybridDns.ps1` を実行すると、上記の双方向 DNS 転送をまとめて設定します。

```powershell
.\infra\network\Setup-HybridDns.ps1
```

スクリプトは以下の 4 ステップを実行します（[3/4] と [4/4] はオプション）。

| ステップ | 方向 | 内容 |
|---|---|---|
| [1/4] | オンプレ → クラウド | DC01 に `privatelink.database.windows.net` → DNS Resolver Inbound IP の条件付きフォワーダーを追加 |
| [2/4] | クラウド → オンプレ | DNS Forwarding Ruleset（`dnsrs-hub`）を作成し、`lab.local` → DC01（`10.0.1.4`）の転送ルールを追加 |
| [3/4] | オンプレ → クラウド VM | Private DNS Zone（`azure.internal`）を作成し、Spoke VM の A レコードを自動登録。DC01 に条件付きフォワーダーを追加（`-EnableCloudVmResolution` 指定時のみ） |
| [4/4] | Spoke VNet リンク | Forwarding Ruleset を Spoke1〜4 VNet にもリンクし、Spoke VM からオンプレの名前解決を可能にする（`-LinkSpokeVnets` 指定時のみ） |

> **オプションスイッチ:**
>
> | スイッチ | 用途 | 使うタイミング |
> |---|---|---|
> | `-EnableCloudVmResolution` | [3/4] を実行し、オンプレからクラウド VM をホスト名で解決可能にする | 移行シナリオで必要になった場合 |
> | `-LinkSpokeVnets` | [4/4] を実行し、Forwarding Ruleset を Spoke1〜4 VNet にもリンク | Spoke VM から `lab.local` を名前解決したいとき |
>
> ```powershell
> # 基本実行（[1/4] + [2/4] のみ）
> .\infra\network\Setup-HybridDns.ps1
>
> # クラウド VM の名前解決も有効化
> .\infra\network\Setup-HybridDns.ps1 -EnableCloudVmResolution
>
> # Spoke VNet リンクも追加
> .\infra\network\Setup-HybridDns.ps1 -LinkSpokeVnets
> ```

---

### 方法 2: 手動で個別に実行

#### [1/4] オンプレ → クラウド方向

DC01 に DNS 条件付きフォワーダーを設定します。

```powershell
# DNS Private Resolver のインバウンド IP を取得
$dnsInboundIp = az dns-resolver inbound-endpoint show `
  --resource-group rg-hub `
  --dns-resolver-name dnspr-hub `
  --name inbound `
  --query "ipConfigurations[0].privateIpAddress" -o tsv

$dnsInboundIp

# DC01 に条件付きフォワーダーを追加
az vm run-command invoke `
  --resource-group rg-onprem `
  --name vm-onprem-ad `
  --command-id RunPowerShellScript `
  --scripts "Add-DnsServerConditionalForwarderZone -Name 'privatelink.database.windows.net' -MasterServers '$dnsInboundIp' -ReplicationScope Forest"
```

> 通常 `10.10.5.4` が割り当てられます（サブネット `10.10.5.0/28` の最初の使用可能 IP）。

#### [2/4] クラウド → オンプレ方向

DNS Forwarding Ruleset を作成し、`lab.local` のクエリを DC01 に転送します。

```powershell
# Outbound Endpoint ID を取得
$outboundEpId = az dns-resolver outbound-endpoint show `
  --resource-group rg-hub `
  --dns-resolver-name dnspr-hub `
  --name outbound `
  --query "id" -o tsv

# Hub VNet ID を取得
$hubVnetId = az network vnet show `
  --resource-group rg-hub `
  --name vnet-hub `
  --query "id" -o tsv

# DNS Forwarding Ruleset を作成
az dns-resolver forwarding-ruleset create `
  --resource-group rg-hub `
  --name dnsrs-hub `
  --location japaneast `
  --outbound-endpoints "[{id:$outboundEpId}]"

# 転送ルールを追加
az dns-resolver forwarding-rule create `
  --resource-group rg-hub `
  --ruleset-name dnsrs-hub `
  --name rule-lab-local `
  --domain-name "lab.local." `
  --forwarding-rule-state Enabled `
  --target-dns-servers "[{ip-address:10.0.1.4,port:53}]"

# ルールセットを Hub VNet にリンク
az dns-resolver vnet-link create `
  --resource-group rg-hub `
  --ruleset-name dnsrs-hub `
  --name link-vnet-hub `
  --id $hubVnetId
```

> **[3/4] と [4/4] について:**  
> これらはオプション機能のため、手動手順は省略しています。必要な場合はスクリプト（`-EnableCloudVmResolution` / `-LinkSpokeVnets`）を使用するか、[Setup-HybridDns.ps1](../../infra/network/onprem/Setup-HybridDns.ps1) のコードを参照してください。

## 動作確認

```powershell
# オンプレ → クラウド: 条件付きフォワーダーの確認
az vm run-command invoke `
  --resource-group rg-onprem `
  --name vm-onprem-ad `
  --command-id RunPowerShellScript `
  --scripts "Get-DnsServerZone -Name 'privatelink.database.windows.net' | Format-List ZoneName,ZoneType,MasterServers"

# クラウド → オンプレ: DNS Forwarding Ruleset の確認
az dns-resolver forwarding-rule list `
  --resource-group rg-hub `
  --ruleset-name dnsrs-hub `
  --query "[].{Name:name, Domain:domainName, State:forwardingRuleState, Target:targetDnsServers[0].ipAddress}" `
  -o table

# Ruleset の VNet リンク確認 (Hub + Spoke)
az dns-resolver vnet-link list `
  --resource-group rg-hub `
  --ruleset-name dnsrs-hub `
  --query "[].{Name:name, VNet:virtualNetwork.id}" `
  -o table
```

期待する出力:

条件付きフォワーダー:

```
ZoneName      : privatelink.database.windows.net
ZoneType      : Forwarder
MasterServers : {10.10.5.4}
```

DNS Forwarding Ruleset:

```
Name            Domain       State    Target
--------------  -----------  -------  --------
rule-lab-local  lab.local.   Enabled  10.0.1.4
```

VNet リンク:

```
Name              VNet
----------------  -------------------------

link-vnet-spoke1  .../virtualNetworks/vnet-spoke1
link-vnet-spoke2  .../virtualNetworks/vnet-spoke2
...
```

## 補足

- DNS Forwarding Ruleset と DC01 の条件付きフォワーダーは Bicep では自動設定されません。VPN 接続確立後にこの手順で設定します。
- `Setup-HybridDns.ps1` は冪等に設計されており、既存の条件付きフォワーダーがある場合は MasterServers を更新します。

> [!TIP]
> Azure CLI と `az vm run-command` で DNS 設定状態・双方向の名前解決を一括検証できるスクリプトも用意しています。
> VM 内部には入らず外部から諸元（Resolver 状態・転送ルール・条件付きフォワーダー・クロスネットワーク名前解決など）を確認する簡易チェックのため、ポータル画面での目視確認は含みません。
>
> ```powershell
> .\infra\network\scripts\Verify-HybridDns.ps1
> ```

> [!TIP]
> **オンプレ ↔ Spoke 間のネットワーク疎通を確認したい場合**
>
> Spoke1（`snet-web`）に一時的な Windows VM を作成し、VPN・Firewall・ピアリング経由の通信が正常かテストできます。
>
> **VM の作成（CLI ワンライナー）**
>
> ```powershell
> az vm create -g rg-spoke1 -n vm-spoke1-web `
>   --image Win2022AzureEditionCore `
>   --size Standard_B2s `
>   --vnet-name vnet-spoke1 --subnet snet-web `
>   --public-ip-address '""' `
>   --admin-username azureuser `
>   --admin-password '<パスワードを指定>' `
>   --os-disk-delete-option Delete --nic-delete-option Delete `
>   --tags Environment=PoC Project=Migration-Handson SecurityControl=Ignore
> ```
>
> > PowerShell では `--public-ip-address ""` が空引数として認識されないため、`'""'` と記述します。
>
> **疎通テスト**
>
> Azure Bastion（`bas-hub`）から `vm-spoke1-web` に RDP 接続し、以下を実行します。
>
> ```powershell
> # Spoke1 → オンプレ (DC01) への疎通確認
> Test-NetConnection -ComputerName 10.0.1.4 -Port 53
> Test-NetConnection -ComputerName 10.0.1.4 -Port 3389
> ```
>
> オンプレ側（APP01）からの確認は、テスト VM の IP を取得してから実行します。
>
> ```powershell
> # テスト VM のプライベート IP を確認
> az vm show -g rg-spoke1 -n vm-spoke1-web -d --query privateIps -o tsv
> ```
>
> **テスト後の削除**
>
> ```powershell
> az vm delete -g rg-spoke1 -n vm-spoke1-web --yes
> az network nsg delete -g rg-spoke1 -n vm-spoke1-webNSG
> ```
>
> `--os-disk-delete-option Delete` と `--nic-delete-option Delete` を作成時に指定しているため、VM 削除で OS ディスクと NIC は自動削除されます。ただし `az vm create` が自動作成する NSG は残るため、別途削除が必要です。
