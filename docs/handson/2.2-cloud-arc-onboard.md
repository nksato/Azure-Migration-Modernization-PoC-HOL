# 2.2. Azure Arc 登録

> **Note**: Azure Arc は、オンプレミスや他クラウドなど **Azure 以外のリソースを Azure の管理・統制モデルに統合する** ためのサービスです。Azure VM 自体は既に Azure の管理下にあるため、通常は Arc の対象にはしません。本ラボでは Azure VM を Arc に登録することで、Arc のアーキテクチャと管理体験を理解することを目的としています。

移行元サーバー `DC01` / `DB01` / `APP01` を **Azure Arc** に登録し、Azure の管理プレーンから一元管理できるようにします。

> 本ラボのサーバーは実体が **Azure VM** のため、通常のオンプレ向けオンボードではなく、  
> [Azure 仮想マシンで Azure Arc 対応サーバーを評価する](https://learn.microsoft.com/ja-jp/azure/azure-arc/servers/plan-evaluate-on-azure-virtual-machine)  
> の手順で登録します。

## 目的

- 移行前サーバーを Azure の管理対象に統合する
- Policy / Monitor / Defender を適用できる状態にする
- オンプレミスも Azure から管理できることを体験する

## 前提条件

- [`2.1-cloud-explore-onprem.md`](./2.1-cloud-explore-onprem.md) が完了している
- `rg-onprem` に対象 VM が存在している
- Azure CLI / PowerShell を利用できる
- Arc 登録先リソース グループに対して必要な権限がある
  - `Azure Connected Machine Onboarding` ロール
  - 必要に応じて `Azure Connected Machine Resource Administrator` または `Contributor`
- VM から Azure Connected Machine Agent を取得できる送信接続がある  
  （制限がある場合は Microsoft Learn の記載に沿い手動配置で対応）

### 事前に準備する値

| パラメータ | 既定値 | 説明 |
|-----------|--------|------|
| `ResourceGroupName` | *(なし — 必須)* | VM が存在するリソースグループ名（例: `rg-onprem`） |
| `ArcResourceGroupName` | `ResourceGroupName` と同じ | Arc リソースを登録するリソースグループ名 |
| `Location` | `japaneast` | Arc リソースのリージョン |
| `VmNames` | `vm-onprem-ad`, `vm-onprem-sql`, `vm-onprem-web` | Arc 対応にする VM 名の配列 |

> `TenantId` / `SubscriptionId` / `ServicePrincipalId` は省略時に現在のコンテキストから自動取得されます。

## 対象サーバー

| サーバー | 役割 | Azure VM リソース名 |
|---|---|---|
| `DC01` | Active Directory / DNS | `vm-onprem-ad` |
| `DB01` | SQL Server | `vm-onprem-sql` |
| `APP01` | Web アプリ | `vm-onprem-web` |

## 推奨手順: `Convert-VmToArc.ps1` を利用

本ラボの VM は Azure VM のため、Arc 登録前にいくつかの準備が必要です（詳細は「手動で実施する場合」を参照）。  
`Convert-VmToArc.ps1` を使うと、これらの準備と登録をまとめて実行できます。

リポジトリに同梱のスクリプトで、上記の準備と登録をまとめて実行できます。

```powershell
Set-Location .\infra\onprem

# 全 VM (vm-onprem-ad / vm-onprem-sql / vm-onprem-web) を Arc 対応にする
.\Convert-VmToArc.ps1 -ResourceGroupName "rg-onprem"

# 特定の VM のみ対象にする場合
.\Convert-VmToArc.ps1 -ResourceGroupName "rg-onprem" -VmNames @("vm-onprem-web")
```

## 手動で実施する場合

スクリプトを使わずに手動で行う場合は、以下の手順を順番に実施します。  
Azure VM に対してそのまま `azcmagent connect` を実行すると、Azure VM として検出され失敗するため、事前準備が必要です。

1. 環境変数 `MSFT_ARC_TEST=true` を設定する
2. VM 拡張機能を削除する
3. `WindowsAzureGuestAgent` サービスを無効化する
4. IMDS（`169.254.169.254` / `169.254.169.253`）へのアクセスをブロックする
5. Azure Connected Machine Agent をインストールする
6. `azcmagent connect` で Azure Arc に登録する

詳細は Microsoft Learn の評価手順を参照してください。

- [Azure 仮想マシンで Azure Arc 対応サーバーを評価する](https://learn.microsoft.com/ja-jp/azure/azure-arc/servers/plan-evaluate-on-azure-virtual-machine)

## 完了確認

```powershell
az connectedmachine list `
  --resource-group rg-onprem `
  --query "[].{name:name,status:status,os:osName}" `
  -o table
```

- Azure Arc のマシン一覧に 3 台が表示される
- ステータスが `Connected` である
- 必要に応じてタグが付与されている

## 備考

### オンボーディングの詳細検証

> [!TIP]
> Bastion に接続せず、ローカル PC から `az connectedmachine run-command` で一括検証できるスクリプトも用意しています。
> VM 内部に入らず外部から諸元（環境変数・サービス状態・エージェント接続状況など）を確認する簡易チェックです。

```powershell
Set-Location .\infra\onprem
.\scripts\Verify-ArcOnboarding.ps1 -ResourceGroupName "rg-onprem"
```

### Arc リソースを別のリソースグループに登録する場合

`-ArcResourceGroupName` パラメータで Arc リソースの登録先を変更できますが、**リソースグループの削除順序に注意が必要です**。Arc リソースが残っている状態で VM 側のリソースグループを先に削除すると、Arc 側のクリーンアップが正常に完了しないことがあります。詳細は [`2.7-cloud-cleanup.md`](./2.7-cloud-cleanup.md) を参照してください。

```powershell
.\Convert-VmToArc.ps1 -ResourceGroupName "rg-onprem" -ArcResourceGroupName "rg-arc"
```

### 通常のオンプレサーバーを Arc 登録する場合

実際のオンプレミスや他クラウドのサーバーでは、本ラボのような Azure VM 向け準備は不要です。  
Azure Portal でオンボーディング スクリプトを生成し、対象サーバー上で Azure Connected Machine Agent をインストール・接続します。
