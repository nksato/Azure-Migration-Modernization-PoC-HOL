# 2.3. ハイブリッド管理

Azure Arc に接続した `DC01` / `DB01` / `APP01` に対して、Azure の管理・監視・セキュリティ機能を適用します。

## 目的

- Azure Monitor Agent (AMA) による監視データ収集
- Log Analytics によるハートビート・パフォーマンス・イベントの可視化
- Azure Update Manager によるパッチ管理
- Azure Policy によるガバナンス確認
- Defender for Cloud によるセキュリティ評価

## 前提条件

- [`2.2-cloud-arc-onboard.md`](./2.2-cloud-arc-onboard.md) が完了し、Arc サーバー 3 台が `Connected` になっている
- `law-hub`（Log Analytics ワークスペース）が `rg-hub` に存在している

---

## 1. Azure Monitor Agent (AMA) のインストール

Arc サーバーに AMA をインストールし、データ収集ルール (DCR) を関連付けます。

### AMA のインストール

```powershell
# 3 台の Arc サーバーに AMA をインストール
$arcMachines = @('vm-onprem-ad-Arc', 'vm-onprem-sql-Arc', 'vm-onprem-web-Arc')

foreach ($machine in $arcMachines) {
    az connectedmachine extension create `
        --resource-group rg-onprem `
        --machine-name $machine `
        --name AzureMonitorWindowsAgent `
        --type AzureMonitorWindowsAgent `
        --publisher Microsoft.Azure.Monitor `
        --no-wait
    Write-Host "AMA install submitted: $machine"
}
```

### データ収集ルール (DCR) の作成

パフォーマンスカウンターと Windows イベントログを Log Analytics に送信する DCR を作成します。

> DCR テンプレート: `infra/tmp/dcr-onprem.json`

```powershell
# DCR を作成
az monitor data-collection rule create `
    --name dcr-onprem-perf-events `
    --resource-group rg-hub `
    --location japaneast `
    --rule-file infra/tmp/dcr-onprem.json

# 3 台の Arc サーバーを DCR に関連付け
$dcrId = az monitor data-collection rule show `
    -g rg-hub --name dcr-onprem-perf-events --query id -o tsv

foreach ($machine in $arcMachines) {
    $arcId = az connectedmachine show `
        -g rg-onprem --name $machine --query id -o tsv

    az monitor data-collection rule association create `
        --name "assoc-$machine" `
        --rule-id $dcrId `
        --resource $arcId
    Write-Host "DCR associated: $machine"
}
```

### 収集データ

| カテゴリ | 内容 |
|---|---|
| パフォーマンス | CPU 使用率、メモリ空き容量、ディスク空き率、ディスク I/O |
| イベントログ | Application / System の Warning 以上 |

## 2. Log Analytics による確認

### ハートビート確認

```powershell
$workspaceId = az monitor log-analytics workspace show `
    -g rg-hub -n law-hub --query customerId -o tsv

az monitor log-analytics query `
    --workspace $workspaceId `
    --analytics-query "Heartbeat | summarize LastHeartbeat=max(TimeGenerated) by Computer | order by Computer asc"
```

### パフォーマンスデータ確認（AMA からのデータ到着後）

```
Perf
| where TimeGenerated > ago(1h)
| summarize avg(CounterValue) by Computer, CounterName
| order by Computer asc, CounterName asc
```

> AMA インストール後、データが Log Analytics に届くまで 5〜10 分程度かかります。

## 3. Azure Update Manager

Azure Update Manager を使用して、Arc サーバーのパッチ状態を確認し、更新を管理します。

### パッチ評価の実行

```powershell
# 各 Arc サーバーのパッチ状態を評価
foreach ($machine in $arcMachines) {
    az connectedmachine assess-patches `
        --resource-group rg-onprem `
        --name $machine `
        --no-wait
    Write-Host "Patch assessment submitted: $machine"
}
```

### Portal での確認

1. Azure Portal → **Azure Update Manager** を開く
2. **マシン** タブで 3 台の Arc サーバーを確認
3. **更新プログラムの状態** で保留中のパッチを確認

### 更新の即時インストール（オプション）

```powershell
# 1 台ずつワンタイム更新を実行 (Critical + Security)
az connectedmachine install-patches `
    --resource-group rg-onprem `
    --name vm-onprem-web-Arc `
    --maximum-duration "PT2H" `
    --reboot-setting IfRequired `
    --windows-parameters "classificationsToInclude=Critical,Security" `
    --no-wait
```

> 更新には時間がかかるため、ハンズオンでは評価のみでも OK です。

Azure Portal で以下を確認します。

## 5. Defender for Cloud（セキュリティ確認）

1. **Defender for Cloud** → **推奨事項** を開く
2. Arc サーバーに関連する推奨事項を確認
3. セキュアスコアを確認
