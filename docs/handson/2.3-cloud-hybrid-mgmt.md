# 2.3. ハイブリッド管理

Azure Arc に接続した `DC01` / `DB01` / `APP01` に対して、Azure Portal から管理・監視・セキュリティ機能を適用します。

## 目的

- Azure Monitor Agent (AMA) による監視データ収集
- Log Analytics によるハートビート・パフォーマンス・イベントの可視化
- Azure Update Manager によるパッチ管理
- Azure Policy によるガバナンス確認
- Defender for Cloud によるセキュリティ評価

## 前提条件

- [`2.2-cloud-arc-onboard.md`](./2.2-cloud-arc-onboard.md) が完了し、Arc サーバー 3 台が `Connected` になっている
- `law-hub`（Log Analytics ワークスペース）が `rg-hub` に存在している

---

## 1. Azure Monitor Agent (AMA) のインストール

### Portal から AMA をインストール

1. Azure Portal → **Azure Arc** → **サーバー** を開く
2. 対象の Arc サーバー（例: `vm-onprem-ad-Arc`）をクリック
3. 左メニュー **拡張機能** → **追加** をクリック
4. **Azure Monitor Windows Agent** を選択 → **次へ** → **作成**
5. 残り 2 台 (`vm-onprem-sql-Arc`, `vm-onprem-web-Arc`) にも同様に実施

> 3 台すべてに AMA をインストールしてください。

### データ収集ルール (DCR) の作成

1. Azure Portal → **モニター** → **データ収集ルール** を開く
2. **作成** をクリック
3. 以下を設定:

| 項目 | 値 |
| --- | --- |
| ルール名 | `dcr-onprem-perf-events` |
| リソース グループ | `rg-hub` |
| リージョン | `Japan East` |

4. **リソース** タブ → **リソースの追加** → 3 台の Arc サーバーを選択
5. **収集と配信** タブ → **データ ソースの追加**:
   - **パフォーマンス カウンター**: CPU / メモリ / ディスクを選択（既定の基本セットで OK）
   - **Windows イベント ログ**: Application と System の Warning 以上を選択
6. **宛先**: `law-hub` (Log Analytics ワークスペース) を選択
7. **確認と作成** → **作成**

---

## 2. Log Analytics でデータを確認

> AMA インストール + DCR 関連付け後、データが届くまで **5〜10 分**程度かかります。

### ハートビート確認

1. Azure Portal → **Log Analytics ワークスペース** → `law-hub` を開く
2. 左メニュー **ログ** をクリック
3. 以下のクエリを実行:

```kql
Heartbeat
| summarize LastHeartbeat=max(TimeGenerated) by Computer
| order by Computer asc
```

**期待結果**: 3 台の Arc サーバー名（DC01, DB01, APP01）が表示される

### パフォーマンスデータ確認

```kql
Perf
| where TimeGenerated > ago(1h)
| summarize avg(CounterValue) by Computer, CounterName
| order by Computer asc, CounterName asc
```

### イベントログ確認

```kql
Event
| where TimeGenerated > ago(1h)
| summarize count() by Computer, EventLog, EventLevelName
| order by Computer asc
```

---

## 3. Azure Update Manager

### パッチ評価

1. Azure Portal → **Azure Update Manager** を開く
2. **概要** で Arc サーバーが表示されることを確認
3. **マシン** タブ → 3 台の Arc サーバーを選択 → **更新プログラムの確認** をクリック
4. 評価が完了すると、保留中のパッチ数が表示される

### 更新の実行（オプション）

1. **マシン** タブ → 更新したいサーバーを選択
2. **1 回限りの更新** をクリック
3. 分類: **Critical** と **Security** を選択
4. 再起動オプション: **必要な場合に再起動** を選択
5. **インストール** をクリック

> 更新には時間がかかるため、ハンズオンでは**評価のみ**でも OK です。

### メンテナンス スケジュール（参考）

定期的な自動パッチ適用を設定したい場合は、**メンテナンス構成** を作成し、Arc サーバーを割り当てます。

---

## 4. Azure Policy（ガバナンス確認）

1. Azure Portal → **Azure Policy** → **コンプライアンス** を開く
2. スコープを `rg-onprem` に設定
3. Arc サーバーに適用されているポリシーの準拠状況を確認
4. 非準拠の項目があれば、原因と推奨アクションを確認

---

## 5. Defender for Cloud（セキュリティ確認）

1. Azure Portal → **Defender for Cloud** を開く
2. **セキュア スコア** で全体のスコアを確認
3. **推奨事項** → フィルターで Arc サーバーを絞り込み
4. 推奨事項の内容と修復手順を確認

---

## 完了確認

- [ ] AMA が 3 台にインストールされている
- [ ] DCR が作成され、Arc サーバーに関連付けられている
- [ ] Log Analytics にハートビートまたはパフォーマンスデータが届いている
- [ ] Azure Update Manager で 3 台のパッチ状態が確認できる
- [ ] Defender for Cloud に推奨事項が表示される

## 備考: CLI で実行する場合

Portal の代わりに CLI で AMA インストール・DCR 作成・パッチ評価を行うこともできます。

<details>
<summary>CLI コマンド例（クリックで展開）</summary>

```powershell
# AMA インストール
$arcMachines = @('vm-onprem-ad-Arc', 'vm-onprem-sql-Arc', 'vm-onprem-web-Arc')
foreach ($machine in $arcMachines) {
    az connectedmachine extension create `
        -g rg-onprem --machine-name $machine `
        --name AzureMonitorWindowsAgent `
        --type AzureMonitorWindowsAgent `
        --publisher Microsoft.Azure.Monitor --no-wait
}

# DCR 作成 (テンプレート: infra/tmp/dcr-onprem.json)
az monitor data-collection rule create `
    --name dcr-onprem-perf-events -g rg-hub `
    --location japaneast --rule-file infra/tmp/dcr-onprem.json

# DCR 関連付け
$dcrId = az monitor data-collection rule show `
    -g rg-hub --name dcr-onprem-perf-events --query id -o tsv
foreach ($machine in $arcMachines) {
    $arcId = az connectedmachine show -g rg-onprem --name $machine --query id -o tsv
    az monitor data-collection rule association create `
        --name "assoc-$machine" --rule-id $dcrId --resource $arcId
}

# パッチ評価
foreach ($machine in $arcMachines) {
    az connectedmachine assess-patches -g rg-onprem --name $machine --no-wait
}

# ハートビート確認
$wsId = az monitor log-analytics workspace show -g rg-hub -n law-hub --query customerId -o tsv
az monitor log-analytics query --workspace $wsId `
    --analytics-query "Heartbeat | summarize max(TimeGenerated) by Computer"
```

</details>
