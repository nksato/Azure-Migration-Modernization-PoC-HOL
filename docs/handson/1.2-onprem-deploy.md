# 1.2. 疑似オンプレ環境のデプロイ

> **Note**  
> [`1.1-initial-setup.md`](./1.1-initial-setup.md) の **Deploy to Azure** でセットアップ済みの場合、このページの手順は不要です。

Azure 上に**移行元として使う疑似オンプレ環境**を構築します。

## 目的

- `DC01` / `DB01` / `APP01` を含む疑似オンプレ環境を Azure 上に作成する
- Azure Bastion 経由の管理アクセスを確保する

## 前提条件

- Azure サブスクリプション
- リソースグループ作成権限
- VM 管理者パスワード

### 事前に準備する値

| パラメータ | 既定値 | 説明 |
|-----------|--------|------|
| `adminUsername` | `labadmin` | VM 管理者ユーザー名 |
| `adminPassword` | *(なし — 必須)* | VM 管理者パスワード（12 文字以上、大文字・小文字・数字・記号を含む） |
| `domainName` | `lab.local` | Active Directory ドメイン名 |

## 使用するテンプレート

| テンプレート | 用途 |
|---|---|
| `infra/onprem/main.bicep` | サブスクリプションスコープ ラッパー (RG 作成 + resources.bicep 呼び出し) |
| `infra/onprem/resources.bicep` | 標準ラボ構成 (リソースグループスコープ) |

## Deploy to Azure ボタン

[![Deploy to Azure](https://aka.ms/deploytoazurebutton)](https://portal.azure.com/#create/Microsoft.Template/uri/https%3A%2F%2Fraw.githubusercontent.com%2Fnksato%2FAzure-Migration-Modernization-PoC-HOL%2Fmain%2Finfra%2Fonprem%2Fmain.json)

> Portal でリージョンとパラメータを入力してデプロイします。リソースグループ `rg-onprem` は自動作成されます。

---

## 方法 1: PowerShell スクリプトでデプロイ（推奨）

`infra/onprem/Deploy-Onprem.ps1` を使うと、管理者パスワードや VPN 共有キーを対話的に入力しながら、再実行しやすくなります。

```powershell
Set-Location .\infra\onprem

# 標準ラボ構成
.\Deploy-Onprem.ps1 `
  -ResourceGroupName "rg-onprem" `
  -Location "japaneast"
```

---

## 方法 2: Bicep を直接デプロイ

```powershell
az deployment sub create `
  --name hol-onprem-base `
  --location japaneast `
  --template-file infra/onprem/main.bicep `
  --parameters adminPassword='<管理者パスワード>'
```

---

## 完了確認

- `vm-onprem-ad` / `vm-onprem-sql` / `vm-onprem-web` が作成されている
- `Azure Bastion` が作成されている
- `NAT Gateway` (`ng-onprem`) が作成されている
- VM 3 台にパブリック IP が付いていない

> VPN Gateway はこのステップでは作成されません。Step 4（[`1.6-cloud-vpn-connect.md`](./1.6-cloud-vpn-connect.md)）で別途デプロイします。
